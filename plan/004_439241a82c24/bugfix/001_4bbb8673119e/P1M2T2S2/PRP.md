## Goal

**Feature Goal**: Stop `npm run format:check` (and therefore `npm run validate`) from
failing on machine-generated pipeline state files by adding `artifacts/` and `plan/`
to `.prettierignore`. Prettier's current glob `"**/*.{ts,js,json,md,yml,yaml}"` sweeps
in generated files (`artifacts/**/checkpoints.json`, `plan/**/tasks.json`,
`plan/**/PRP.md`, etc.) which are written by the pipeline, not hand-formatted, and so
they perpetually fail the `--check`.

**Deliverable**: A modified `.prettierignore` file containing all 6 existing entries
unchanged, plus two new entries on their own lines: `artifacts/` and `plan/`. No other
file is modified.

**Success Definition**: `npx prettier --check "**/*.{ts,js,json,md,yml,yaml}"` no longer
reports any file whose path begins with `artifacts/` or `plan/`. Equivalently,
`npm run format:check` exits 0 *with respect to the generated-state class of failures*
(remaining failures, if any, must be hand-authored source files — out of scope for this
task).

## User Persona (if applicable)

**Target User**: Pipeline maintainer / developer running `npm run validate`.

**Use Case**: Running the project's validation gate (`npm run validate` =
`lint && format:check && typecheck`) during local development or CI without it
failing on files the developer did not author and cannot meaningfully reformat.

**User Journey**:
1. Developer (or CI) runs `npm run validate`.
2. `format:check` step no longer emits `[warn]` lines for `artifacts/**` or `plan/**`.
3. The gate proceeds to `typecheck` without a false failure from generated state.

**Pain Points Addressed**: Generated pipeline artifacts (checkpoints, tasks.json,
PRP snapshots) are written by tooling in a format Prettier considers unformatted, so
every `format:check` run fails for reasons unrelated to the developer's changes. This
makes the green-gate signal (the whole point of Issue 4 / P1.M2.T2) unreliable.

## Why

- **Unblocks the `npm run validate` gate** that every Session-004 task contract
  requires ("`npm run validate` must pass"). `validate` chains `lint && format:check
  && typecheck`; a failing `format:check` fails the whole gate.
- **Correctly scopes Prettier's responsibility**: Prettier should enforce formatting on
  *hand-authored* source (`src/`, `tests/`, `docs/`, `scripts/`), not on
  *machine-generated pipeline state* that is rewritten on every run and is never edited
  by humans.
- **Companion to P1.M2.T2.S1** (the ToolExecutor type-drift fix, already Complete):
  together they restore `npm run validate` to green. This task owns *only* the
  `format:check`-on-generated-state slice; the typecheck slice is owned by S1.

## What

Append exactly two lines — `artifacts/` and `plan/`, each on its own line — to the end
of `.prettierignore`. Preserve all 6 existing entries verbatim and in their current
order. Do not reorder, dedupe, comment, or reformat existing lines.

### Scope Boundaries (HARD — do not exceed)

- **DO NOT** exclude `docs/` or `tests/` — those are hand-authored and MUST remain
  subject to Prettier.
- **DO NOT** change the `format`/`format:check` glob in `package.json` — the fix
  belongs in `.prettierignore` (the contract specifies this), not in the script.
- **DO NOT** create or modify any other file (no new config, no `.prettierrc` change,
  no touching `tsconfig*`, `package.json`, or any source file).
- **DO NOT** modify `PRD.md`, any `tasks.json`, any `prd_snapshot.md`, or `.gitignore`
  (these are orchestrator/human-owned; see FORBIDDEN OPERATIONS).
- **DO NOT** attempt to *reformat* the generated files as an alternative — they are
  regenerated by the pipeline and would just fail again on the next run.

### Success Criteria

- [ ] `.prettierignore` contains the original 6 entries **and** `artifacts/` **and**
      `plan/` (8 entries total).
- [ ] `npx prettier --check "**/*.{ts,js,json,md,yml,yaml}"` emits zero `[warn]` lines
      whose path begins with `artifacts/` or `plan/`.
- [ ] `npm run format:check` no longer fails *because of* `artifacts/`/`plan/` files.
- [ ] No existing `.prettierignore` entry is removed or altered.
